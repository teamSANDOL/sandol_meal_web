# Sandol Meal Web

`sandol_meal_web`는 산돌이 학식 서비스의 관리자/식당 주인용 기능성 웹 UI를 제공하는 FastAPI 기반 BFF 서비스입니다.

## 목표

- Keycloak 로그인 세션 관리
- `sandol_meal_service` API를 서버 사이드에서 호출
- 디자인보다 기능 동작을 우선한 서버 렌더링 HTML 제공

## 실행

```bash
uv sync
uv run uvicorn main:app --host 0.0.0.0 --port 5800 --reload
```

## 프론트엔드 자산

UI는 서버 렌더링 HTML 위에 정적 자산으로 구성되어 있습니다.

- CSS/JS 위치: `app/static/`
- FastAPI 정적 경로: `/meal-web/static/...`
- 별도 Node.js/Next.js 실행은 필요하지 않습니다.

## 현재 상태

- Keycloak 인증/세션 기반 서버 렌더링 UI 제공
- 식당주 등록 요청 및 관리자 검토 흐름 제공

## 관리자/식당 주인 사용 가이드

외부 접속 기준 기본 경로는 `/meal-web` 입니다.

### 공통 흐름

1. `/meal-web/`에 접속합니다.
2. 루트 페이지에서 `식당주 바로가기` 또는 `관리자 바로가기`를 클릭합니다.
3. 로그인이 안 되어 있으면 Keycloak 로그인 페이지로 이동합니다.
4. 로그인 완료 후 다시 `meal-web`로 돌아옵니다.
5. 로그인 후 화면 상단의 `로그아웃` 버튼으로 로그아웃할 수 있습니다.

### 식당 주인(owner) 사용 흐름

1. `/meal-web/`에서 `식당주 바로가기`를 클릭합니다.
2. `식당 등록 요청 목록` 화면에서 기존 요청 목록을 확인합니다.
3. `새 식당 등록 요청`을 눌러 등록 폼으로 이동합니다.
4. 식당명, 식당 유형, 위치, 운영 시간 등을 입력하고 `등록 요청 제출`을 누릅니다.
5. 등록 후 상세 화면 또는 목록 화면에서 요청 상태를 확인합니다.
6. 상세 화면에서 제출 시간, 검토 시간, 거절 사유를 확인할 수 있습니다.
7. 필요하면 상세 화면에서 삭제 확인 체크 후 `삭제`를 눌러 요청을 제거합니다.

### 관리자(admin) 사용 흐름

1. `/meal-web/`에서 `관리자 바로가기`를 클릭합니다.
2. `관리자 식당 등록 요청 목록` 화면에서 전체 요청을 확인합니다.
3. 각 요청의 `보기`를 눌러 상세 화면으로 이동합니다.
4. 상세 화면에서 식당 정보, 위치, 시간, 거절 사유를 검토합니다.
5. 문제가 없으면 `승인`을 눌러 요청을 승인합니다.
6. 반려해야 하면 거부 사유를 입력한 뒤 `거부`를 눌러 요청을 거부합니다.

### 참고

- 비로그인 사용자가 보호된 페이지에 접근하면 로그인 페이지로 이동합니다.
- 관리자 페이지는 `global_admin` 또는 `meal_admin` 권한이 필요합니다.
- 모든 상태 변경 요청은 CSRF 보호가 적용됩니다.
